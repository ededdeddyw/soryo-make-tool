# ② 自治体のごみルール ―― 段階的絞り込みフロー 設計メモ

最終更新：2026-06-22 ／ ステータス：設計（未実装）

## 0. 狙い
今の ② は「区を選ぶ→粗大ごみルールを表示」という独立モジュール。これを **①（これどうする？）で得た情報を引き継ぎ、「あなたの場合こう出せる」をまず結論で見せ、知りたい人だけ素材で絞り込み、最終的に“いつ・どう出すか”までシンプルに教える** 設計に変える。

キモは **段階開示（Progressive Disclosure）**：最初は1行の結論。深掘りは任意。

## 1. 全体フロー
```
①ウィザード（状態・サイズ等）
        │ 回答を引き継ぐ（size など）＋ 区（②で選択 or 収集日リマインドのlocalStorageから復元）
        ▼
②デフォルト判定（追加入力なしで即出す）
   「最長辺◯cmなら “粗大ごみ” か？」を 区の基準と照合
   ├─ 粗大ではない → 「粗大ごみではないので“ふつうのごみ”として出せます（無料）。もっと詳しく知りたい？」
   ├─ 粗大かも     → 「粗大ごみの可能性。申込・有料。詳しく見る？」
   └─ 判定できない  → 「この区は袋基準など。公式で確認 ＋ 素材から判定する？」
        │ ［もっと詳しく知りたい / 素材から判定する］を押す
        ▼
③絞り込み（素材などを選ばせる）
   素材：燃やすごみ／不燃・金属／プラ／資源(缶・びん・ペット)／古紙／危険ごみ(電池・スプレー・刃物 等)
        │
        ▼
④最終結論（1枚で完結）
   「これは “一般の不燃（金属）ごみ” だね。
    港区の○○地区は【△曜】回収。前の夜〜当日の朝（多くは8:00まで）に出そう。
    マンション等にお住まいの方は管理規約に従ってね。」
   ＋ 収集日が分かる区・地区なら 🔔 リマインド登録へ
```

## 2. 例（ユーザー提示のシナリオ）
- ① で「最長辺30cm以内」だけ判明 → ② で居住区ルールと照合し **まず粗大ごみから除外**。
- ② デフォルト表示：「**粗大ごみではないので無料でごみとして出せるよ。もっと詳しく知りたい？**」
- ［知りたい］→ 素材を選ばせる →
- 結論：「**粗大ごみではない一般の金属ごみだね。港区では【○曜】回収らしいから、前夜〜当日朝に出そう。マンション等は規約に従ってね。**」
  - ※「水曜」は説明用の仮。実際は区・地区の収集日データから実際の曜日を出す（港区の不燃は地区ごとに第2・4◯曜などのため）。

## 3. 入力ソース（どこから情報を取るか）
| 情報 | 取得元 |
|---|---|
| サイズ（最長辺の目安） | ① wizard `answers.size`（small=〜30cm/ mid=30〜50/ large=50〜） |
| 状態・価格など | ① wizard `answers`（判定の補助。捨てる確定なら主に size を使う） |
| 区（市区町村） | ② のセレクト。未選択なら 収集日リマインドの `localStorage('shobun-navi.area')` から復元を試みる |
| 地区（町・丁目） | 同上（収集日データがある区＝品川・港 等で曜日を出すのに使用） |
| 素材・区分 | ③ の追加質問でユーザーに選ばせる |

## 4. 判定ロジック
### 4-1. 粗大ごみか否か（デフォルト判定）
区の粗大基準は `TOKYO23.basis` / `CITY_RULES[区].basis` にある（ただし表現がバラバラ：「一辺30cm以上」「50cm以上」「指定袋に入らない」等）。
- 区の基準が **cm閾値**（例：港区/東京23区=おおむね30cm以上、横浜=金属30/その他50 等）→ ①の最長辺目安と比較：
  - 目安 < 閾値 → **粗大でない**（無料の一般ごみ）
  - 目安 ≥ 閾値 → **粗大**（申込・有料）
  - 30〜50 など閾値またぎ → **「粗大かも」**（断定しない＋公式リンク）
- 区の基準が **袋基準**（札幌・千葉・神戸・福岡 等「指定袋に入らない大きさ」）→ cmで断定せず **「袋に入るか？」を1問** 出す or 公式確認に誘導（推測しない方針）。
- 信頼度バッジ（conf=high/medium/low）が low/medium の区は表現を弱める。

### 4-2. 素材 → 区分 → 出し方
ユーザーが選んだ素材を、区の分別区分にマッピング：
| 素材選択 | 一般的な区分 | 注意 |
|---|---|---|
| 紙くず・生ごみ・布 | 燃やすごみ（可燃） | |
| 金属・ガラス・陶器 | 不燃ごみ（区により「陶器・ガラス・金属ごみ」） | |
| プラ製品・容器 | プラ／資源プラ（区による） | |
| 缶・びん・ペット | 資源 | |
| 新聞・雑誌・段ボール | 古紙・資源 | |
| 電池・スプレー缶・ライター・刃物・小型家電 | **危険ごみ／特別な出し方** | 必ず警告表示。区で方法が違う |

→ 区分が決まったら、**収集日データ（data/gomi/＜区コード＞.json）** から該当区分の曜日を引いて「○曜回収」を出す。データが無い区分・区は「公式で確認」。

### 4-3. 最終メッセージのテンプレ
```
これは「{区分}」だね。
{区}{地区がある場合は「の◯◯」} では【{曜日/第N◯曜}】回収。
収集日の前の夜〜当日の朝（多くの自治体で 8:00 まで）に、決められた場所へ。
{危険ごみなら：※これは危険ごみ。区の特別な出し方を必ず確認して}
マンション等にお住まいの方は、建物の管理規約・集積所ルールに従ってね。
```
＋ 収集日が確定できた区・地区は「🔔 この収集日をカレンダーに登録」(reminder.html) への導線。

## 5. UI / 状態
- ② はカード内で 3 段階を出し分け（折りたたみ）：**[結論] → [もっと詳しく(素材選択)] → [最終結論]**。
- ①未回答でも ② 単体で使える（区＋サイズ＋素材を手動で選べる）よう、①連携はあくまで“初期値”。
- ①からの遷移時に size と（あれば）区を引き継いで、いきなりデフォルト判定を表示。

## 6. 既存資産との接続
- `TOKYO23` / `CITY_RULES`（基準・手数料・申込・URL）… 粗大判定と公式リンク。
- `data/gomi/13109.json`(品川) / `13103.json`(港区) … 素材→区分→**実際の収集曜日**。今後の区追加でカバー拡大。
- `reminder.html` … 最終結論から収集日リマインド登録へ送客（.ics/カレンダー）。
- ① wizard `answers` … size 等の初期値。

## 7. 方針・注意（重要）
- **推測値を載せない**：基準が袋基準でcm判定できない区は断定せず質問 or 公式誘導。収集日データが無い区分は「公式で確認」。
- 危険ごみ（電池・スプレー・刃物・小型家電）は必ず特別扱いの警告。
- マンション・集合住宅の独自ルールは必ず免責。
- 祝日・年末年始の収集休みは収集日データに含まれない旨を明記（既存の注記を流用）。
